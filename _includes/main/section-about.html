JAM - JavaScript + API + Markup

The JAMstack is not about specific technologies. Itâ€™s a new way of building websites and apps that delivers better performance, higher security, lower cost of scaling, and a better developer experience.

Pre-rendered sites can be enhanced with JavaScript and the growing capabilities of browsers and services available via APIs.

Static sites might be built using sites built by hand, or with Jekyll, Hugo, Nuxt, Next, Gatsby, or another static site generator.

If you can use it on the web, you can probably build it with the Jamstack. Millions already do.

API
Better Performance - pre-built files

Why wait for pages to build on the fly when you can generate them at deploy time?

Higher Security - no server-side processes

With server-side processes abstracted into microservice APIs, surface areas for attacks are reduced.

Cheaper - Easier Scaling

When your deployment amounts to a stack of files that can be served anywhere, scaling is a matter of serving those files in more places.
